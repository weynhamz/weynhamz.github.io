Projects I Created建立的项目

  • dotploy

    A bash script to deploy $HOME dot files across different users and hosts.

    一个依据主机名和用户名部署 dot files 的 Bash 脚本。

    As a system administrator, I often manage several Linux boxes, each one of them have a set of dot files under our $HOME directory. Some of these dot files actually are shared among, there should be a way to be able to manage all these dot files in a central location and deploy them per user per host basis. After searching the Internet for a while I couldn't find a way that satisfy me. From my point of view, I think tools that do these kind of work are better writing in Shell scripts, or at least in Perl. So, I wrote my own. dotploy is a script is a bash script written in pure Bash, discard the POSIX standard, as bash is actually the default shell in many distros. With dotploy, all we need to do is maintain a central dot files repo, and distribute it to the target hosts with dropbox or VCS tools, then everything is taken care of.

    我经常需要维护多个 Linux 主机，每个都有自己的一组 dot files，而其中的大部分是在多个主机之间通用的。我想应该有办法可以集中的管理这些文件，然后根据主机名和用户名进行部署，然而，我并没有找到能够让我满意的方案。在我看来，做这件事情工具最好是使用 Shell 脚本写的，或者至少是 Perl 脚本。于是，我造了我自己的轮子。dotploy 使用 Bash 写成，并不严格遵守 POSIX 规范，因为 Bash 在很多发行版上都是默认的 Shell。通过维护一个 dot files 的中央仓库，然后使用 dropbox 或者任何 VSC 工具同步到目标主机，剩下的工作 dotploy 自会搞定。

  • PHPCtags

    An enhanced ctags compatible index generator written in pure PHP.

    一个 PHP 写的 ctags 兼容的索引生成器。

    Ctags is a very useful tool for programmers, but the support for PHP is limited. As a PHP developer, I have the desire for a better tags file with scope and access support. As a result, instead of patching the current Ctags' C code, I decide to write a new tags generator written in pure PHP, this is how this project was born. It uses PHP_Parser as PHP syntax parsing backend. The tags index file it generated is compatible with ctags' fromat.

    Ctags对于程序员来说是一个非常有用的工具，但是它对 PHP 的支持却非常有限。作为一个 PHP 程序员，我需要一个包含作用域和访问控制信息的 tags 文件，经过一番研究，与其给 Ctags 的 C 代码打补丁，还不如使用 PHP 从头写一个，于是 PHPCtags 就这么诞生了。它使用 PHP_Parser 做为词法分析器，能够产生兼容 ctags 格式的 tags 索引文件。

  • tagbar-phpctags.vim

    An addon for Jan Larres's tagbar Vim plugin that makes use of my PHPCtags to generate tags index for PHP file.

    一个 Jan Larres 写的 tagbar Vim 插件的补充，使用 PHPCtags 生成的 tags 索引文件。

  • Lapurd

    Lapurd is a experiment project of mine that aims to build a Drupal like modular content management framework with modern PHP technologies.The project is in its very early stage and under heavily developing and refactoring, the latest work is at t branch.

    Lapurd 是一个我个人的实验性项目，目的使用最新的 PHP 技术构建一个类似与 Drupal 的模块化的内容管框架。项目目前还处于早期极端，还会有大量的重构，最新的工作位于 t 分支。

Projects I Maintaining维护的项目

  • minibufexpl.vim

    A Vim plugin that shows buffers list in a tab-like fashion with status info.

    一个用来以标签页的形式现实缓冲区列表和状态的插件。

    It was first published as minibufexpl.vim by Bindu Wavell at the end of the year 2001 and last updated by him at the end of the year 2004. Since then, as it is such a useful plugin, many volunteers continued improving it, hence here came Oliver Uvman's minibufexplorer++. After Oliver, Federico Holgado took the torch, he made a lot improvements in his fork on github.

    minibufexpl.vim 最早是在 2001 年末的时候由 Bindu Wavell 发布,最后一次被原作者更新是 2004 年末。从那以后，作为一款非常有用的 Vim 插件，许多志愿者进行了不同程度的改进，比如 Oliver Uvman 的 minibufexplorer++. 在 Oliver 之后, Federico Holgado 在他的 Gibhub Fork 上做了大量的改进。

    In the year 2012, after Federico's fork kept no activity for a while, I decided to keep improving this great plugin, so here comes my fork of minibufexpl.vim. After contacting Federico, I also have the management right of his repository. I released version 6.5.0 on Jun 16, 2013 in which almost every known bugs ever reported have been fixed, details are on the milestones of 6.5.0 ([1], [2]).

    2012 年, Federico 的 fork 有段时间没有动静了, 我决定继续改进这款插件，于是有了我的 Fork. 在联系过 Federico 之后，我也拥有了他的仓库的提交权限。我在 2013 年 6 月 16 日释出了版本 6.5.0，释出之时基本上每一个已知的 bug 都得到了修复，详细可见里程碑 6.5.0 ([1], [2])。
